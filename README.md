# Iron Log — встановлення на iPhone

Цей додаток — повністю автономний PWA (Progressive Web App). Він не з App Store, тому встановлюється напряму через Safari. Дані зберігаються прямо на телефоні (в локальному сховищі Safari) і нікуди не відправляються через інтернет.

## Крок 1. Розмісти файли там, де Safari зможе їх відкрити по http(s)

PWA-функції (інсталяція іконки, офлайн-режим, service worker) **не працюють** напряму з файлової системи (`file://`). Потрібен будь-який звичайний веб-хостинг із усією цією папкою. Найпростіші безкоштовні варіанти:

### Варіант A — GitHub Pages (рекомендовано, безкоштовно назавжди)
1. Створи акаунт на github.com, якщо ще немає.
2. Створи новий публічний репозиторій (наприклад `iron-log`).
3. Завантаж туди всі файли з цієї папки (`index.html`, `app.jsx`, `manifest.json`, `sw.js`, `icon-192.png`, `icon-512.png`, папку `vendor/`) — просто перетягни їх через веб-інтерфейс GitHub (кнопка "Add file → Upload files").
4. У репозиторії: Settings → Pages → Source: вибери гілку `main`, папку `/ (root)` → Save.
5. За хвилину сайт стане доступним за адресою на кшталт `https://твій-юзернейм.github.io/iron-log/`.

### Варіант B — Netlify Drop (ще простіше, без акаунта)
1. Відкрий https://app.netlify.com/drop
2. Перетягни цю всю папку в браузер.
3. Отримаєш посилання типу `https://random-name-123.netlify.app` — готово.

## Крок 2. Встанови на головний екран iPhone

1. Відкрий отримане посилання в **Safari** (обов'язково Safari, не Chrome — на iOS тільки Safari вміє встановлювати PWA).
2. Натисни кнопку "Поділитися" (квадрат зі стрілкою вгору) внизу екрана.
3. Прокрути список дій вниз і вибери **"На екран Початку"** (Add to Home Screen).
4. Підтверди назву (за замовчуванням "Iron Log") → Додати.

Готово — на головному екрані з'явиться іконка з гантеллю. Відкривається вона як звичайний додаток на весь екран, без адресного рядка Safari.

## Як зберігаються дані

Всі тренування, рекорди, заміри тіла та щоденник їжі зберігаються в `localStorage` Safari — це постійне сховище, прив'язане саме до того сайту (домену), з якого ти встановив додаток. Дані **залишаються на телефоні** і переживають:
- закриття і повторне відкриття додатку,
- перезавантаження телефону,
- режим польоту / відсутність інтернету (додаток працює офлайн завдяки `sw.js`).

### Важливо
- Не видаляй сам сайт-хостинг (GitHub-репозиторій / Netlify-сайт) — він потрібен лише один раз при першому встановленні; після цього додаток працює офлайн, **але** якщо колись очистиш дані сайту в Safari (Settings → Safari → Advanced → Website Data → знайти і видалити) — це зітре і тренування.
- Дані не синхронізуються між пристроями автоматично (це локальне сховище одного телефону). Якщо захочеш бекап чи синхронізацію між iPhone і комп'ютером — це окрема функція, яку можна додати пізніше (наприклад, експорт/імпорт у файл).

## Оновлення додатку в майбутньому

Якщо я внесу зміни в код (нові функції, виправлення), потрібно:
1. Замінити файли на хостингу (GitHub / Netlify) новими версіями.
2. В `sw.js` зміни рядок `const CACHE_NAME = "iron-log-v1";` на `"iron-log-v2"` (і так далі) — це змусить Safari підвантажити свіжу версію замість закешованої.
3. Просто перевідкрий додаток на телефоні — оновиться автоматично, дані (localStorage) при цьому не зачепляться.
